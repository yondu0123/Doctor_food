# ⚙️ 환경 설정

> DoctorFood 환경 변수 및 설정 가이드

---

## 📋 개요

DoctorFood는 환경 변수를 통해 다양한 설정을 관리합니다. 개발, 스테이징, 프로덕션 환경에 따라 다른 설정을 사용할 수 있습니다.

## 🔧 환경 변수 파일

### 1. 백엔드 (.env)

```bash
# backend/.env

# ===== 데이터베이스 =====
DATABASE_URL=sqlite:///./doctorfood.db
# PostgreSQL 사용 시:
# DATABASE_URL=postgresql://user:password@localhost:5432/doctorfood

# ===== JWT 인증 =====
SECRET_KEY=your-super-secret-key-change-this-in-production-min-32-chars
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=10080  # 7일 (60*24*7)

# ===== 카카오 OAuth =====
KAKAO_CLIENT_ID=your_kakao_rest_api_key
KAKAO_CLIENT_SECRET=your_kakao_client_secret
KAKAO_REDIRECT_URI=http://localhost:8000/api/auth/kakao/callback

# ===== Google Gemini AI =====
GEMINI_API_KEY=your_gemini_api_key_here

# ===== 서버 설정 =====
HOST=0.0.0.0
PORT=8000
DEBUG=True
RELOAD=True

# ===== CORS 설정 =====
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:5500,http://127.0.0.1:5500

# ===== 파일 업로드 =====
UPLOAD_DIR=./uploads
MAX_UPLOAD_SIZE=10485760  # 10MB (bytes)
ALLOWED_EXTENSIONS=jpg,jpeg,png,gif

# ===== 로깅 =====
LOG_LEVEL=INFO
LOG_FILE=./logs/app.log
```

### 2. 프론트엔드 (config.js)

```javascript
// frontend/shared/js/config.js

const CONFIG = {

```

## 🔐 환경 변수 상세 설명

### 1. 데이터베이스 (DATABASE_URL)

**SQLite (개발용)**
```bash
DATABASE_URL=sqlite:///./doctorfood.db
```

**PostgreSQL (프로덕션 권장)**
```bash
DATABASE_URL=postgresql://username:password@localhost:5432/doctorfood
```

**MySQL**
```bash
DATABASE_URL=mysql+pymysql://username:password@localhost:3306/doctorfood
```

---

### 2. JWT 인증

#### SECRET_KEY
- JWT 토큰 서명에 사용되는 비밀 키
- **최소 32자 이상** 권장
- 절대 공개하지 말 것!

**생성 방법:**
```bash
# Python으로 생성
python -c "import secrets; print(secrets.token_urlsafe(32))"

# OpenSSL로 생성
openssl rand -base64 32
```

#### ALGORITHM
- JWT 서명 알고리즘
- 기본값: `HS256` (HMAC-SHA256)

#### ACCESS_TOKEN_EXPIRE_MINUTES
- JWT 토큰 만료 시간 (분 단위)
- 개발: `10080` (7일)
- 프로덕션: `1440` (1일) 권장

---

### 3. 카카오 OAuth

#### KAKAO_CLIENT_ID
- 카카오 개발자 콘솔에서 발급받은 REST API 키
- [카카오 개발자 콘솔](https://developers.kakao.com/)에서 확인

#### KAKAO_CLIENT_SECRET
- 카카오 앱의 Client Secret (선택사항)
- 보안 강화를 위해 설정 권장

#### KAKAO_REDIRECT_URI
- 카카오 로그인 후 리다이렉트될 URI
- 카카오 개발자 콘솔에 등록된 URI와 **정확히 일치**해야 함

**예시:**
```bash
# 개발
KAKAO_REDIRECT_URI=http://localhost:8000/api/auth/kakao/callback

# 프로덕션
KAKAO_REDIRECT_URI=https://api.doctorfood.com/api/auth/kakao/callback
```

---

### 4. Google Gemini AI

#### GEMINI_API_KEY
- Google AI Studio에서 발급받은 API 키
- [Google AI Studio](https://makersuite.google.com/app/apikey)에서 발급

**발급 방법:**
1. Google AI Studio 접속
2. "Get API Key" 클릭
3. 프로젝트 선택 또는 생성
4. API 키 복사

---

### 5. 서버 설정

#### HOST
- 서버가 바인딩될 호스트
- `0.0.0.0`: 모든 네트워크 인터페이스
- `127.0.0.1`: 로컬호스트만

#### PORT
- 서버 포트 번호
- 기본값: `8000`

#### DEBUG
- 디버그 모드 활성화
- 개발: `True`
- 프로덕션: `False`

#### RELOAD
- 코드 변경 시 자동 재시작
- 개발: `True`
- 프로덕션: `False`

---

### 6. CORS 설정

#### ALLOWED_ORIGINS
- CORS 허용 오리진 목록 (쉼표로 구분)
- 프론트엔드 URL을 모두 포함해야 함

**예시:**
```bash
# 개발
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:5500,http://127.0.0.1:5500

# 프로덕션
ALLOWED_ORIGINS=https://doctorfood.com,https://www.doctorfood.com
```

---

### 7. 파일 업로드

#### UPLOAD_DIR
- 업로드된 파일이 저장될 디렉토리
- 상대 경로 또는 절대 경로

#### MAX_UPLOAD_SIZE
- 최대 업로드 파일 크기 (바이트)
- 10MB = `10485760`

#### ALLOWED_EXTENSIONS
- 허용할 파일 확장자 (쉼표로 구분)
- 예: `jpg,jpeg,png,gif`

---

### 8. 로깅

#### LOG_LEVEL
- 로그 레벨
- 옵션: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`

#### LOG_FILE
- 로그 파일 경로
- 예: `./logs/app.log`

## 📝 설정 파일 (config.py)

```python
# backend/app/core/config.py
from pydantic_settings import BaseSettings
from typing import List

class Settings(BaseSettings):
    """애플리케이션 설정"""
    
    # 데이터베이스
    DATABASE_URL: str = "sqlite:///./doctorfood.db"
    
    # JWT
    SECRET_KEY: str
    ALGORITHM: str = "HS256"
    ACCESS_TOKEN_EXPIRE_MINUTES: int = 10080
    
    # 카카오 OAuth
    KAKAO_CLIENT_ID: str
    KAKAO_CLIENT_SECRET: str = ""
    KAKAO_REDIRECT_URI: str
    
    # Gemini AI
    GEMINI_API_KEY: str
    
    # 서버
    HOST: str = "0.0.0.0"
    PORT: int = 8000
    DEBUG: bool = True
    RELOAD: bool = True
    
    # CORS
    ALLOWED_ORIGINS: str = "http://localhost:3000"
    
    @property
    def allowed_origins_list(self) -> List[str]:
        """CORS 허용 오리진 리스트로 변환"""
        return [origin.strip() for origin in self.ALLOWED_ORIGINS.split(",")]
    
    # 파일 업로드
    UPLOAD_DIR: str = "./uploads"
    MAX_UPLOAD_SIZE: int = 10485760  # 10MB
    ALLOWED_EXTENSIONS: str = "jpg,jpeg,png,gif"
    
    @property
    def allowed_extensions_list(self) -> List[str]:
        """허용 확장자 리스트로 변환"""
        return [ext.strip() for ext in self.ALLOWED_EXTENSIONS.split(",")]
    
    # 로깅
    LOG_LEVEL: str = "INFO"
    LOG_FILE: str = "./logs/app.log"
    
    class Config:
        env_file = ".env"
        case_sensitive = True

# 싱글톤 인스턴스
settings = Settings()
```

### 3. 환경 변수 검증

```python
# backend/app/core/config.py
from pydantic import validator

class Settings(BaseSettings):
    SECRET_KEY: str
    
    @validator('SECRET_KEY')
    def secret_key_must_be_strong(cls, v):
        if len(v) < 32:
            raise ValueError('SECRET_KEY must be at least 32 characters')
        return v
    
    @validator('GEMINI_API_KEY')
    def gemini_api_key_required(cls, v):
        if not v:
            raise ValueError('GEMINI_API_KEY is required')
        return v
```

## 🐛 문제 해결

### 1. "SECRET_KEY not found" 오류
```bash
# .env 파일이 올바른 위치에 있는지 확인
ls -la backend/.env

# 환경 변수 직접 설정
export SECRET_KEY="your-secret-key"
```

### 2. 카카오 로그인 실패
```bash
# Redirect URI 확인
echo $KAKAO_REDIRECT_URI

# 카카오 개발자 콘솔의 설정과 일치하는지 확인
```

### 3. CORS 오류
```bash
# ALLOWED_ORIGINS에 프론트엔드 URL 추가
ALLOWED_ORIGINS=http://localhost:5500,http://127.0.0.1:5500
```
