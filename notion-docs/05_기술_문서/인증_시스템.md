# 🔐 인증 시스템

> DoctorFood JWT 인증 및 OAuth 통합 가이드

## 📋 개요

DoctorFood는 **JWT (JSON Web Token)** 기반 인증과 **카카오 OAuth 2.0** 소셜 로그인을 지원합니다.

### 지원하는 인증 방식
1. **이메일/비밀번호** - 전통적인 회원가입 및 로그인
2. **카카오 OAuth** - 카카오 계정으로 간편 로그인
3. **(예정) 네이버 OAuth** - 네이버 계정 로그인

## 🔑 JWT 인증

### 1. JWT 토큰 구조

```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxIiwiZW1haWwiOiJ1c2VyQGV4YW1wbGUuY29tIiwiZXhwIjoxNzE0ODI0MDAwfQ.signature
```

**구성 요소:**
- **Header**: 알고리즘 정보 (HS256)
- **Payload**: 사용자 정보 (user_id, email, exp)
- **Signature**: 서명 (SECRET_KEY로 생성)

### 2. 토큰 생성

```python
# backend/app/auth/jwt.py
from datetime import datetime, timedelta
from jose import jwt
from ..core.config import settings

def create_access_token(data: dict):
    """JWT 액세스 토큰 생성"""
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    
    encoded_jwt = jwt.encode(
        to_encode,
        settings.SECRET_KEY,
        algorithm=settings.ALGORITHM
    )
    return encoded_jwt
```

**Payload 예시:**
```json
{
  "sub": "1",
  "email": "user@example.com",
  "role": "caregiver",
  "exp": 1714824000
}
```

### 3. 토큰 검증

```python
# backend/app/auth/dependencies.py
from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from jose import JWTError, jwt

security = HTTPBearer()

async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security),
    db: Session = Depends(get_db)
):
    """JWT 토큰에서 현재 사용자 추출"""
    token = credentials.credentials
    
    try:
        payload = jwt.decode(
            token,
            settings.SECRET_KEY,
            algorithms=[settings.ALGORITHM]
        )
        user_id: str = payload.get("sub")
        if user_id is None:
            raise credentials_exception
    except JWTError:
        raise credentials_exception
    
    user = db.query(User).filter(User.id == int(user_id)).first()
    if user is None:
        raise credentials_exception
    
    return user
```

## 🔐 비밀번호 해싱

### 1. 비밀번호 암호화

```python
# backend/app/auth/jwt.py
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def get_password_hash(password: str) -> str:
    """비밀번호를 bcrypt로 해싱"""
    return pwd_context.hash(password)

def verify_password(plain_password: str, hashed_password: str) -> bool:
    """비밀번호 검증"""
    return pwd_context.verify(plain_password, hashed_password)
```

**예시:**
```python
# 회원가입 시
hashed = get_password_hash("mypassword123")
# → "$2b$12$KIXxJ5vZ..."

# 로그인 시
is_valid = verify_password("mypassword123", hashed)
# → True
```

### 3. Authorization Code 처리

```python
# backend/app/services/kakao_auth.py
import httpx
from ..core.config import settings

class KakaoAuthService:
    def __init__(self):
        self.client_id = settings.KAKAO_CLIENT_ID
        self.client_secret = settings.KAKAO_CLIENT_SECRET
        self.redirect_uri = settings.KAKAO_REDIRECT_URI
    
    async def get_access_token(self, code: str) -> str:
        """Authorization Code로 Access Token 발급"""
        url = "https://kauth.kakao.com/oauth/token"
        data = {
            "grant_type": "authorization_code",
            "client_id": self.client_id,
            "client_secret": self.client_secret,
            "redirect_uri": self.redirect_uri,
            "code": code
        }
        
        async with httpx.AsyncClient() as client:
            response = await client.post(url, data=data)
            response.raise_for_status()
            return response.json()["access_token"]
    
    async def get_user_info(self, access_token: str) -> dict:
        """Access Token으로 사용자 정보 조회"""
        url = "https://kapi.kakao.com/v2/user/me"
        headers = {"Authorization": f"Bearer {access_token}"}
        
        async with httpx.AsyncClient() as client:
            response = await client.get(url, headers=headers)
            response.raise_for_status()
            return response.json()
```

### 4. 카카오 로그인 API

```python
# backend/app/api/auth.py
@router.post("/kakao/login", response_model=Token)
async def kakao_login(
    oauth_data: OAuthLogin,
    db: Session = Depends(get_db),
    kakao_service: KakaoAuthService = Depends(get_kakao_auth_service)
):
    """카카오 OAuth 로그인"""
    
    # 1. Authorization Code로 Access Token 발급
    access_token = await kakao_service.get_access_token(oauth_data.code)
    
    # 2. Access Token으로 사용자 정보 조회
    kakao_user = await kakao_service.get_user_info(access_token)
    
    # 3. 기존 사용자 확인 또는 신규 생성
    oauth_id = str(kakao_user["id"])
    user = db.query(User).filter(
        User.oauth_provider == "kakao",
        User.oauth_id == oauth_id
    ).first()
    
    if not user:
        # 신규 사용자 생성
        user = User(
            email=kakao_user["kakao_account"]["email"],
            username=f"kakao_{oauth_id}",
            full_name=kakao_user["kakao_account"]["profile"]["nickname"],
            role=oauth_data.role,
            oauth_provider="kakao",
            oauth_id=oauth_id,
            profile_image=kakao_user["kakao_account"]["profile"]["profile_image_url"]
        )
        db.add(user)
        db.commit()
        db.refresh(user)
    
    # 4. JWT 토큰 생성
    jwt_token = create_access_token(
        data={"sub": str(user.id), "email": user.email, "role": user.role}
    )
    
    return {
        "access_token": jwt_token,
        "token_type": "bearer",
        "user": user
    }
```

## 🛡️ 권한 관리 (Role-Based Access Control)

### 1. 사용자 역할

```python
# backend/app/models/user.py
class UserRole(str, enum.Enum):
    ADMIN = "admin"        # 시설 관리자
    CAREGIVER = "caregiver"  # 간병인
    GUARDIAN = "guardian"    # 보호자
```

### 2. 역할별 권한 체크

```python
# backend/app/auth/dependencies.py
from fastapi import HTTPException, status

def require_role(allowed_roles: list[UserRole]):
    """특정 역할만 접근 가능하도록 제한"""
    def role_checker(current_user: User = Depends(get_current_user)):
        if current_user.role not in allowed_roles:
            raise HTTPException(
                status_code=status.HTTP_403_FORBIDDEN,
                detail="권한이 없습니다"
            )
        return current_user
    return role_checker
```

### 3. 역할별 엔드포인트 보호

```python
# 관리자만 접근 가능
@router.delete("/patients/{patient_id}")
```

## 🔒 보안 모범 사례

### 1. 토큰 저장
- ✅ **권장**: `localStorage` 또는 `sessionStorage`
- ❌ **비권장**: 쿠키 (CSRF 공격 위험)

```javascript
// 토큰 저장
localStorage.setItem('access_token', token);

// 토큰 사용
const token = localStorage.getItem('access_token');
fetch('/api/meals/', {
    headers: {
        'Authorization': `Bearer ${token}`
    }
});
```

### 2. HTTPS 사용
- 프로덕션 환경에서는 반드시 HTTPS 사용
- HTTP에서는 토큰이 평문으로 전송됨

### 3. 토큰 만료 시간
- 짧은 만료 시간 설정 (7일 권장)
- Refresh Token 도입 고려

### 4. 비밀번호 정책
- 최소 8자 이상
- 영문, 숫자, 특수문자 조합 권장
- bcrypt로 해싱 (cost factor 12)

### 5. Rate Limiting
- 로그인 시도 제한 (5회/분)
- 무차별 대입 공격 방지

## 🐛 문제 해결

### 1. "Invalid token" 오류
- 토큰 만료 확인
- SECRET_KEY 일치 여부 확인
- 토큰 형식 확인 (Bearer 포함)

### 2. 카카오 로그인 실패
- Redirect URI 일치 여부 확인
- 카카오 개발자 콘솔에서 앱 설정 확인
- Client ID/Secret 확인

### 3. 권한 오류 (403 Forbidden)
- 사용자 역할 확인
- 엔드포인트 권한 설정 확인
