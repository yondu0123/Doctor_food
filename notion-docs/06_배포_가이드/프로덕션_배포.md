## 📋 개요

이 문서는 DoctorFood를 실제 프로덕션 환경에 배포하는 방법을 설명합니다. Docker를 사용한 컨테이너 배포와 전통적인 서버 배포 방법을 모두 다룹니다.

## 🐳 Docker를 사용한 배포 (권장)

### 1. Dockerfile 작성

#### 백엔드 Dockerfile

```dockerfile
# backend/Dockerfile
FROM python:3.11-slim

# 작업 디렉토리 설정
WORKDIR /app

# 시스템 패키지 업데이트
RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# Python 패키지 설치
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 애플리케이션 코드 복사
COPY . .

# 업로드 디렉토리 생성
RUN mkdir -p uploads logs

# 포트 노출
EXPOSE 8000

# 서버 실행
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

#### 프론트엔드 Dockerfile

```dockerfile
# frontend/Dockerfile
FROM nginx:alpine

# 프론트엔드 파일 복사
COPY . /usr/share/nginx/html

# Nginx 설정 복사
COPY nginx.conf /etc/nginx/conf.d/default.conf

# 포트 노출
EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 2. docker-compose.yml

```yaml
# docker-compose.yml
version: '3.8'

services:
  # PostgreSQL 데이터베이스
  db:
    image: postgres:15-alpine
    container_name: doctorfood-db
    environment:
      POSTGRES_DB: doctorfood
      POSTGRES_USER: doctorfood_user
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - doctorfood-network
    restart: unless-stopped

  # 백엔드 API
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: doctorfood-backend
    environment:
      DATABASE_URL: postgresql://doctorfood_user:${DB_PASSWORD}@db:5432/doctorfood
      SECRET_KEY: ${SECRET_KEY}
      GEMINI_API_KEY: ${GEMINI_API_KEY}
      KAKAO_CLIENT_ID: ${KAKAO_CLIENT_ID}
      KAKAO_CLIENT_SECRET: ${KAKAO_CLIENT_SECRET}
      KAKAO_REDIRECT_URI: ${KAKAO_REDIRECT_URI}
      DEBUG: "False"
      ALLOWED_ORIGINS: ${ALLOWED_ORIGINS}
    volumes:
      - ./backend/uploads:/app/uploads
      - ./backend/logs:/app/logs
    ports:
      - "8000:8000"
    depends_on:
      - db
    networks:
      - doctorfood-network
    restart: unless-stopped

  # 프론트엔드
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: doctorfood-frontend
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./ssl:/etc/nginx/ssl:ro
    networks:
      - doctorfood-network
    restart: unless-stopped

volumes:
  postgres_data:

networks:
  doctorfood-network:
    driver: bridge
```

### 3. 배포 실행

```bash
# 1. 프로젝트 클론
git clone https://github.com/your-username/doctorfood-app.git
cd doctorfood-app

# 2. 환경 변수 설정
cp .env.example .env
nano .env  # 프로덕션 값으로 수정

# 3. Docker 이미지 빌드 및 실행
docker-compose up -d --build

# 4. 로그 확인
docker-compose logs -f

# 5. 상태 확인
docker-compose ps
```

### 4. Docker 관리 명령어

```bash
# 컨테이너 중지
docker-compose down

# 컨테이너 재시작
docker-compose restart

# 특정 서비스만 재시작
docker-compose restart backend

# 로그 확인
docker-compose logs -f backend

# 컨테이너 내부 접속
docker-compose exec backend bash

# 데이터베이스 백업
docker-compose exec db pg_dump -U doctorfood_user doctorfood > backup.sql

# 데이터베이스 복원
docker-compose exec -T db psql -U doctorfood_user doctorfood < backup.sql
```

## 🖥️ 전통적인 서버 배포

### 1. 서버 준비

```bash
# Ubuntu 서버 업데이트
sudo apt update && sudo apt upgrade -y

# 필수 패키지 설치
sudo apt install -y python3.11 python3.11-venv python3-pip nginx postgresql postgresql-contrib git

# 방화벽 설정
sudo ufw allow 22    # SSH
sudo ufw allow 80    # HTTP
sudo ufw allow 443   # HTTPS
sudo ufw enable
```

### 2. PostgreSQL 설정

```bash
# PostgreSQL 접속
sudo -u postgres psql

# 데이터베이스 및 사용자 생성
CREATE DATABASE doctorfood;
CREATE USER doctorfood_user WITH PASSWORD 'your_secure_password';
GRANT ALL PRIVILEGES ON DATABASE doctorfood TO doctorfood_user;
\q
```

### 3. 애플리케이션 배포

```bash
# 애플리케이션 디렉토리 생성
sudo mkdir -p /var/www/doctorfood
sudo chown $USER:$USER /var/www/doctorfood

# 프로젝트 클론
cd /var/www/doctorfood
git clone https://github.com/your-username/doctorfood-app.git .

# 백엔드 설정
cd backend
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 환경 변수 설정
cp .env.example .env
nano .env  # 프로덕션 값으로 수정

# 데이터베이스 초기화
python -c "from app.core.database import init_db; init_db()"
```

### 4. Systemd 서비스 설정

```bash
# Systemd 서비스 파일 생성
sudo nano /etc/systemd/system/doctorfood.service
```

```ini
[Unit]
Description=DoctorFood FastAPI Application
After=network.target postgresql.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/var/www/doctorfood/backend
Environment="PATH=/var/www/doctorfood/backend/venv/bin"
ExecStart=/var/www/doctorfood/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 서비스 활성화 및 시작
sudo systemctl daemon-reload
sudo systemctl enable doctorfood
sudo systemctl start doctorfood

# 상태 확인
sudo systemctl status doctorfood
```

### 5. Nginx 설정

```bash
# Nginx 설정 파일 생성
sudo nano /etc/nginx/sites-available/doctorfood
```

```nginx
# HTTP → HTTPS 리다이렉트
server {
    listen 80;
    server_name doctorfood.com www.doctorfood.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS 서버
server {
    listen 443 ssl http2;
    server_name doctorfood.com www.doctorfood.com;

    # SSL 인증서
    ssl_certificate /etc/letsencrypt/live/doctorfood.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/doctorfood.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 프론트엔드
    location / {
        root /var/www/doctorfood/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 백엔드 API
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 업로드 파일
    location /uploads/ {
        alias /var/www/doctorfood/backend/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 파일 업로드 크기 제한
    client_max_body_size 10M;
}
```

```bash
# 설정 활성화
sudo ln -s /etc/nginx/sites-available/doctorfood /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 💾 백업 전략

### 1. 데이터베이스 백업

```bash
# 수동 백업
docker-compose exec db pg_dump -U doctorfood_user doctorfood > backup_$(date +%Y%m%d).sql

# 자동 백업 스크립트 (cron)
sudo nano /usr/local/bin/backup-db.sh
```

```bash
#!/bin/bash
BACKUP_DIR="/var/backups/doctorfood"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR
docker-compose exec -T db pg_dump -U doctorfood_user doctorfood > $BACKUP_DIR/backup_$DATE.sql

# 7일 이상 된 백업 삭제
find $BACKUP_DIR -name "backup_*.sql" -mtime +7 -delete
```

```bash
# 실행 권한 부여
sudo chmod +x /usr/local/bin/backup-db.sh

# Cron 설정 (매일 새벽 2시)
sudo crontab -e
0 2 * * * /usr/local/bin/backup-db.sh
```

### 2. 파일 백업

```bash
# 업로드 파일 백업
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz backend/uploads/

# 원격 서버로 전송 (rsync)
rsync -avz backend/uploads/ user@backup-server:/backups/doctorfood/uploads/
```

## 🐛 문제 해결

### 1. 서비스가 시작되지 않음

```bash
# 로그 확인
sudo journalctl -u doctorfood -n 50

# 포트 충돌 확인
sudo lsof -i :8000

# 권한 확인
ls -la /var/www/doctorfood
```

### 2. 데이터베이스 연결 실패

```bash
# PostgreSQL 상태 확인
sudo systemctl status postgresql

# 연결 테스트
psql -h localhost -U doctorfood_user -d doctorfood
```

### 3. 502 Bad Gateway

```bash
# 백엔드 서비스 확인
sudo systemctl status doctorfood

# Nginx 설정 테스트
sudo nginx -t

# 로그 확인
sudo tail -f /var/log/nginx/error.log
```
