## 🔧 환경 변수 파일 구조

```
backend/
├── .env                    # 실제 환경 변수 (Git 제외)
├── .env.example           # 예시 파일 (Git 포함)
```

## 📝 .env.example (템플릿)

```bash
# ===== 데이터베이스 =====

# ===== JWT 인증 =====
SECRET_KEY=change-this-to-a-random-secret-key-min-32-chars
ALGORITHM=HS256
```

## 🔐 환경 변수 상세 설명

### 1. DATABASE_URL

**형식:**
```
[데이터베이스]://[사용자명]:[비밀번호]@[호스트]:[포트]/[데이터베이스명]
```

**예시:**
```bash
# SQLite (개발)
DATABASE_URL=sqlite:///./doctorfood.db

# PostgreSQL (프로덕션)
DATABASE_URL=postgresql://user:pass@localhost:5432/doctorfood

# MySQL
DATABASE_URL=mysql+pymysql://user:pass@localhost:3306/doctorfood
```

---

### 2. SECRET_KEY

**생성 방법:**

```bash
# Python으로 생성 (권장)
python -c "import secrets; print(secrets.token_urlsafe(32))"

# OpenSSL로 생성
openssl rand -base64 32

# Node.js로 생성
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

**예시 출력:**
```
xK7vN2mP9qR4sT6uW8yZ1aB3cD5eF7gH9iJ0kL2mN4oP6qR8sT0uV2wX4yZ6aB8c
```

**주의사항:**
- 최소 32자 이상
- 환경마다 다른 키 사용
- 절대 공개하지 말 것
- 정기적으로 변경 (6개월~1년)

---

### 3. KAKAO_CLIENT_ID & KAKAO_CLIENT_SECRET

**발급 방법:**

1. [카카오 개발자 콘솔](https://developers.kakao.com/) 접속
2. "내 애플리케이션" → "애플리케이션 추가하기"
3. 앱 이름 입력 후 생성
4. "앱 키" 탭에서 REST API 키 복사 → `KAKAO_CLIENT_ID`
5. "보안" 탭에서 Client Secret 활성화 및 복사 → `KAKAO_CLIENT_SECRET`
6. "플랫폼" 탭에서 Redirect URI 등록

**Redirect URI 설정:**
```bash
# 개발
http://localhost:8000/api/auth/kakao/callback

# 프로덕션
https://api.doctorfood.com/api/auth/kakao/callback
```

---

### 4. GEMINI_API_KEY

**발급 방법:**

1. [Google AI Studio](https://makersuite.google.com/app/apikey) 접속
2. Google 계정으로 로그인
3. "Get API Key" 클릭
4. 프로젝트 선택 또는 생성
5. API 키 복사

**예시:**
```bash
GEMINI_API_KEY=AIzaSyD1234567890abcdefghijklmnopqrstuvwxyz
```

---

### 5. ALLOWED_ORIGINS

**형식:**
```bash
ALLOWED_ORIGINS=origin1,origin2,origin3
```

**예시:**
```bash
# 개발 (여러 포트)
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:5500,http://127.0.0.1:5500

# 프로덕션 (HTTPS만)
ALLOWED_ORIGINS=https://doctorfood.com,https://www.doctorfood.com

# 와일드카드 (비권장)
ALLOWED_ORIGINS=*
```

**주의사항:**
- 프로덕션에서는 와일드카드(`*`) 사용 금지
- HTTPS 사용 권장
- 불필요한 오리진 제거

## 🔄 환경 변수 전환

### 1. 수동 전환

```bash
# 개발 환경으로 전환
cp .env.development .env

# 스테이징 환경으로 전환
cp .env.staging .env

# 프로덕션 환경으로 전환
cp .env.production .env
```

### 2. 스크립트를 사용한 전환

```bash
# scripts/switch-env.sh
#!/bin/bash

ENV=$1

if [ -z "$ENV" ]; then
    echo "Usage: ./switch-env.sh [development|staging|production]"
    exit 1
fi

if [ ! -f ".env.$ENV" ]; then
    echo "Error: .env.$ENV not found"
    exit 1
fi

cp ".env.$ENV" .env
echo "Switched to $ENV environment"
```

```bash
# 사용법
chmod +x scripts/switch-env.sh
./scripts/switch-env.sh development
```

## ✅ 환경 변수 검증

### 1. Python 스크립트로 검증

```python
# scripts/validate_env.py
import os
from dotenv import load_dotenv

load_dotenv()

required_vars = [
    'DATABASE_URL',
    'SECRET_KEY',
    'GEMINI_API_KEY',
    'KAKAO_CLIENT_ID',
]

missing_vars = []

for var in required_vars:
    if not os.getenv(var):
        missing_vars.append(var)

if missing_vars:
    print("❌ Missing environment variables:")
    for var in missing_vars:
        print(f"  - {var}")
    exit(1)
else:
    print("✅ All required environment variables are set")

# SECRET_KEY 길이 확인
secret_key = os.getenv('SECRET_KEY')
if len(secret_key) < 32:
    print("⚠️  WARNING: SECRET_KEY should be at least 32 characters")
```

```bash
# 실행
python scripts/validate_env.py
```

### 2. 서버 시작 시 자동 검증

```python
# backend/app/core/config.py
from pydantic import validator

class Settings(BaseSettings):
    SECRET_KEY: str
    
    @validator('SECRET_KEY')
    def validate_secret_key(cls, v):
        if len(v) < 32:
            raise ValueError('SECRET_KEY must be at least 32 characters')
        return v
```

```python
import boto3

def get_secret(secret_name):
    client = boto3.client('secretsmanager')
    response = client.get_secret_value(SecretId=secret_name)
```
